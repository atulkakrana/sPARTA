<h2><a id="user-content-sparta" class="anchor" href="#sparta" aria-hidden="true"><span class="octicon octicon-link"></span></a>sPARTA</h2>

<p><b>s</b>mall RNA-<b>PA</b>RE <b>T</b>arget <b>A</b>nalyzer<br> 
Updated: version-1.11 4/1/2015</p>

<h2><a id="user-content-description" class="anchor" href="#description" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Description</b></h2>

<p>small RNA-PARE Target Analyzer (sPARTA) is a tool which utilizes
high-throughput sequencing to profile genome-wide cleavage products.
sPARTA begins with a built-in parallelized target prediction module for plant
miRNAs called <code>miRferno</code>. sPARTA as a whole utilizes multi-core servers to
achieve two-dimensional parallelization in order to maintain a low memory
footprint, imperative to achieve a full genome analysis. </p>

<h2><a id="user-content-dependencies" class="anchor" href="#dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Dependencies</b></h2>

<p><b>sPARTA</b> requires bowtie2 in the PATH variable of the user account executing sPARTA<br>


<code>bowtie2</code> may be downloaded here <a href="http://bowtie-bio.sourceforge.net/bowtie2/index.shtml">http://bowtie-bio.sourceforge.net/bowtie2/index.shtml</a><br></p>

<p>sPARTA requires the following python3 functions to perform properly:<br>
<code>numpy</code> - <a href="http://www.numpy.org/">http://www.numpy.org/</a><br>
<code>pyfasta</code> - <a href="https://pypi.python.org/pypi/pyfasta/">https://pypi.python.org/pypi/pyfasta/</a><br>
<code>rpy2</code> - <a href="http://rpy.sourceforge.net/">http://rpy.sourceforge.net/</a><br>
<code>scipy</code> - <a href="http://www.scipy.org/">http://www.scipy.org/</a><br>
<br>
These may easily be installed using (Python) <code>PIP</code>. Intructions to install<code>PIP</code> - <a href="https://pip.pypa.io/en/stable/installing.html">https://pip.pypa.io/en/stable/installing.html</a><br></p>

<h2><a id="user-content-note" class="anchor" href="#note" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Note</b></h2>

<p>

1.sPARTA uses file extensions to identify file types, naming meta-data and selectively cleaning up temp files. Therefore, it is recommended to have appropriate file extensions.
For Ex. a genome/cDNA FASTA file should have <code>.fa</code> extension.
Please see 'Arguments' section (below) for recommended file extensions.</p>

<p>2.Make sure that input fasta files do not have integers in name. For ex - test.1.fa or arabidopsis.new.2.4.fa
Files with such names are deleted sometimes while cleanup operation</p>

<p>3.All the input files 1) <code>miRNAs</code> 2)<code>FASTA</code> file for genome or transcripts and 3) <code>degradome/PARE</code> in tag-count format should be in same directory, including sPARTA script

</p>

<h2><a id="user-content-execution" class="anchor" href="#execution" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Execution</b></h2>

<p>There are command line arguments that are to be used by sPARTA for proper
execution. For the first execution, all steps must be performed, but
once this has been completed, provided the miRNAs and genome are the same,
the entire analysis will not need to be repeated.  <a href="https://github.com/sdeepti/atulChange/blob/master/README.md#examples">Examples</a> of such cases
may be seen below.</p>

<h2><a id="user-content-arguments" class="anchor" href="#arguments" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Arguments</b></h2><br>
<table>
<tbody><tr>
<td>-gffFile</td>        
<td>GFF3 file for the species being analyzed corresponding  to the genome assembly being used. Recommended file
 extension - '.gff' or '.gff3'</td>
</tr>
<tr>
<td>-genomeFile</td>      
<td>Genome file in FASTA format that will be used to extract features (genic or intergenic regions) using GFF3 file.
Recommended file extension - '.fa'</td>
</tr>
<tr><td>-featureFile</td>
<td>FASTA file containing sequences of interest (CDS, transcript,
 intergenic regions etc.) if user already has a set of
 sequences. This option is mutually exclusive to genome file and
gff file. So either genomefile along with gffFile is used or
 feature set is supplied directly. Recommended file extension - '.fa'</td>
</tr>
<tr><td width="180">-genomeFeature</td>
<td>0 if prediction is to be done in genic region. 1 if prediction
 is to be done in intergenic region</td>
</tr>
<tr><td>-miRNAFile</td><td> FASTA format of miRNA sequences. Recommended file extension - '.fa'</td></tr>
<tr><td>-tarPred</td><td>Mode of target prediction. H for heuristic. E for exhaustive.
 H is default if no mode is specified</td></tr>
<tr><td>-tarScore</td><td>Scoring mode for target prediction. S for seedless. N for
normal. S is default if no mode is specified</td></tr>
<tr><td>-libs</td><td> List of PARE library files in tag count format. Data can be
 easily converted into tag count format using <a href="http://www.ebi.ac.uk/~stijn/reaper/tally.html" target="_blank">tally</a></td></tr>
<tr><td>-tagLen</td>

<tr><td>-accel</td> 
<td>Y to use balanced multiple process scheme or else specify the
             number of processors to be used. Y is default</td></tr>
<tr><td>--tag2FASTA</td>  
<td>Convert tag count file for PARE libraries to FASTA files for
mapping</td></tr>
<tr><td>--map2DD</td>
<td> Map the PARE reads to feature set</td>
</tr><tr><td>--validate</td> 
<td>Flag to perform the validation of the potential cleave sites
 from miRferno</td></tr>
<tr><td>--repeats</td>      
<td> Flag to include PARE reads from repetitive regions</td></tr>
<tr><td>--noiseFilter</td>
<td> Flag to include all PARE validations with p-value of &lt;=.5,
             irrespective of the noise to signal ratio at cleave site and
         category of PARE read.</td></tr>
</tbody></table>

<h2><a id="user-content-genome-and-annotation-data" class="anchor" href="#genome-and-annotation-data" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Genome and Annotation Data</b></h2>

<p>Both the <code>GFF3</code> file and corresponding genome <code>FASTA</code> file can be downloaded from
Phytozome [<a href="http://www.phytozome.net/">http://www.phytozome.net/</a>]<br></p>

<h2><a id="user-content-examples" class="anchor" href="#examples" aria-hidden="true"><span class="octicon octicon-link"></span></a><b>Examples</b></h2>

<p>

1.Execution on new genome/entirely new dataset
This execution should be performed any time a new genome file (along with corresponding <code>GFF</code> file) is being analyzed:</p>

<pre><code>python3 sPARTA.py -genomeFile &lt;genomeFile.fa&gt; -gffFile &lt;GFF3file&gt; -genomeFeature &lt;0/1&gt; -miRNAFile &lt;miRNAFile.fa&gt; -libs &lt;Lib_A.txt Lib_B.txt&gt; -tarPred -tarScore --tag2FASTA --map2DD --validate
</code></pre>

<p>OR</p>

<p>a user provided feature set (FASTA file with sequences of interest) is being analyzed:</p>

<pre><code>python3 sPARTA.py -featureFile &lt;featureFile.fa&gt; -genomeFeature &lt;0/1&gt; -miRNAFile &lt;miRNAFile.fa&gt; -libs &lt;Lib_A.txt Lib_B.txt&gt; -tarPred -tarScore --tag2FASTA --map2DD --validate
</code></pre>

<p>2.Execution on genome in which genome has already been processed
This execution should be performed if a genome file has been processed previously but the miRNAs for which targets need to be predicted are new:</p>

<pre><code>python3 sPARTA.py -genomeFeature &lt;0/1&gt; -miRNAFile &lt;miRNAFile.fa&gt; -libs &lt;Lib_A.txt Lib_B.txt&gt; -tarPred -tarScore --tag2FASTA --map2DD --validate
</code></pre>

<p>3.Execution on data in which genome and miRNA files have been previously processed
This execution should be performed if targets for a genome file have already been predicted using a miRNA file, but new PARE libraries need to be used for validation of earlier predicted targets:</p>

<pre><code>python3 sPARTA.py -genomeFeature &lt;0/1&gt; -libs &lt;Lib_C.txt Lib_D.txt&gt; --map2DD --validate
</code></pre>

<p>4.Execution of 'miRferno', just for target prediction
This execution should be performed in case only predicted targets are required or PARE libraries are not available:</p>

<pre><code>python3 sPARTA.py -genomeFile &lt;genomeFile.fa&gt; -gffFile &lt;GFF3file&gt; -genomeFeature &lt;0/1&gt; -miRNAFile &lt;miRNAFile.fa&gt; -tarPred -tarScore
</code></pre>

<p>OR</p>

<p>a user provided feature set (FASTA file with sequences of interest) is being analyzed:</p>

<pre><code>python3 sPARTA.py -featureFile &lt;featureFile.fa&gt; -genomeFeature &lt;0/1&gt; -miRNAFile &lt;miRNAFile.fa&gt; -tarPred -tarScore
</code></pre>
